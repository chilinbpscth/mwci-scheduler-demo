# 多元智能課排課系統（v6.41）開發者指南 / 設計筆記

> 呢份文件用嚟「記錄原作者嘅設計意圖 + 接手者可以點樣完善」。  
> 目前專案形態係 **單一 HTML 檔**（`卍慈多元智能課排課系統v6.41(with開發者資料).html`），無後端、無資料庫。

---

## 1) 系統定位（想解決咩問題）

- **目標**：為「多元智能課」做排課，輸出一份易讀、可交付嘅 Excel 排課表，同時平衡老師工作量、處理專科專教、避免疲勞（連續工作過多日）。
- **使用場景**：
  - 老師/主任喺瀏覽器直接開檔使用
  - 透過 `.xlsx` 做「設定匯入」同「結果匯出」
  - 可以載入「過往排課」作為歷史節數基準，令新一輪排課更平均

---

## 2) 技術架構（而家點樣砌出嚟）

### 2.1 單檔前端

- **UI/狀態**：Vue 3（`createApp({ setup() { ... } })`）
- **樣式**：Tailwind CDN
- **Excel**：ExcelJS（讀/寫 `.xlsx`）
- **下載**：FileSaver（`saveAs()`）

### 2.2 重要限制（由架構帶出）

- **依賴網絡**：Vue/Tailwind/ExcelJS/FileSaver/字體都係 CDN；無網絡會白屏或功能壞。
- **無持久化**：除咗匯入/匯出 Excel，同一輪操作資料只喺記憶體；重新整理就無。
- **可維護性**：所有邏輯混埋同一個 HTML，擴充時容易變長、變難測。

---

## 3) 介面流程（Step-by-step）

系統 UI 係 4 個 step：

- **Step 1：班級與教師**
  - 班級：輸入班名（逗號/換行分隔），會由班名抽出年級（`P{level}`）
  - 教師：
    - 教師代號（通常 1 字/2 字；亦支援用「A,B」代表第1/2節不同人）
    - 任教班別：主科 / 非主科（用於偏好排序）
    - 專長：用於「主題關鍵字」匹配，做專科專教優先

- **Step 2：日期選擇**
  - 預設產生日期區間，並只揀星期一/星期五（見 `generateDates()`）

- **Step 3：主題與顏色**
  - 每個「日期 x 班別」可輸入：
    - 主題文字（課堂名稱）
    - 背景色（匯出 Excel 時用）
    - 固定老師（可留空＝自動；`/`＝表示不排？；`TBD`＝衝突/待處理）
  - 特別任務（modal）：每個日期可新增任務，設定參與人數、每人負荷計法（全/第1節/第2節/不計）

- **Step 4：排課結果**
  - 生成排課
  - 匯出 Excel（包含歷史＋新排）

> **關鍵 Gate**：由 Step 3 去 Step 4 之前會檢查所有欄位，如果仍然有 `TBD` 會阻止生成，迫使用者先修正。

## 4) 核心資料結構（接手改算法前必讀）

> 呢度係「心臟位」：你要改規則/加功能，大多都係圍繞呢幾個結構。

- `rawClasses: string`：原始班級輸入字串
- `parsedClasses: { name: string, level: number }[]`：由班名推斷年級（`P{level}`）
- `teachers: Teacher[]`：
  - `name: string`
  - `assignments: Record<className, 'main' | 'sub'>`
  - `specialties: string[]`
- `topics: Record<\"${dateVal}_${className}\", { text: string, color: string, fixedTeacher: string }>`
- `specialTasks: Record<dateVal, Task[]>`
  - `Task.assignments: { teacher: string, loadType: 'full'|'1'|'2'|'0' }[]`
- `historySchedule: any[]`：由「過往紀錄」匯入（只用於合併匯出＋歷史節數/疲勞）
- `baseTeacherLoad: Record<teacherName, number>`：歷史節數（匯入過往紀錄時計）
- `teacherLoad: Record<teacherName, number>`：新一輪排課節數（生成排課時計）
- `schedule: any[]`：生成結果（每個日期一個 row object，內有各班欄位）

## 5) 老師代號字串規則（非常重要，牽涉衝突檢查與節數計算）

系統用 `getTeacherSlots(str)` 去解讀「一個 cell 入面嘅老師字串」：

- 空／`TBD`：視為無效
- 包含 `,` / `，` / `/`：
  - 視為「第1節/第2節不同人」，例如 `趙,嫻`
- 否則：
  - 若字串長度 > 1：視為「兩位老師共同上」，例如 `趙嫻`
  - 若字串長度 = 1：視為「同一位老師上兩節」，例如 `趙`

> **注意**：呢個設計對「老師代號係 2 個字（真姓名縮寫）」可能會誤判；完善系統時要優先處理（見 Roadmap）。

## 6) 排課算法（v6.41 既規則與優先序）

主函數：`generateSchedule()`

### 6.1 忙碌檢查（避免同日撞堂）

每個日期會建立 `busySlots[teacherName] = {1: boolean, 2: boolean}`，用嚟標記第1/2節已被佔用。

### 6.2 固定老師（最先落地）

如果某班某日 `fixedTeacher` 唔係空／唔係 `TBD`，就會鎖定：

- 更新 `busySlots`（第1/2節）
- 更新 `teacherLoad`（每班基礎 2 節；共享則拆 1+1）

### 6.3 特別任務（第二優先）

每個特別任務可以：

- **指定老師**：若撞 `busySlots`，會標成 `老師(衝突)`；否則計入節數並標 busy
- **自動分配**：由可用老師中揀，排序（由優先到次要）：
  - 節數較少優先
  - 疲勞較低優先（避免「已連續工作 4 日」嘅人再做第 5 日）
  - 其餘用 random 做 tie-break

### 6.4 班別自動分配（第三優先）

對每班：

1. 先篩「唔忙」老師
2. 如果有主題文字，會用 `specialties` 做匹配（主題包含某專長字串）
3. 候選池優先：
   - 有專長匹配 + 同班任教（main/sub）
   - 有專長匹配
   - 同班任教（main/sub）
   - 任意可用老師
4. 排序（由優先到次要）：
   - 節數較少優先（平衡負荷）
   - 同班任教優先
   - 疲勞較低優先（避免連續 5 日）
   - random tie-break

### 6.5 疲勞定義（連續工作）

`getTeacherStreak(tName)` 會由「歷史 + 已生成結果」向後倒數，計連續有上堂/有參與任務嘅日數；若 streak ≥ 4，就視為「再排落去會變第 5 日」，因此排序時會盡量放後。

## 7) Excel I/O（文件格式契約）

### 7.1 設定範本（downloadTemplate）

檔名：`排課系統_設定範本.xlsx`

- Sheet：`班級設定`
  - A1：班級名稱
  - A2+：每行一個班名
- Sheet：`教師設定`
  - 欄位：`教師代號 / 任教班別 / 性質 / 專長(逗號分隔)`
  - 同一位老師可以多行（每行一個班別 assignment）

### 7.2 載入設定（uploadConfig）

會覆蓋：

- 班級清單
- 教師清單、任教班別、主/非主、專長

### 7.3 載入過往紀錄（uploadHistory）

目的：

- 合併匯出（歷史 + 今次）
- 計 `baseTeacherLoad`，令新一輪排課可以「連同歷史」做均衡

### 7.4 匯出結果（exportToExcel）

檔名：`排課結果_v6.41.xlsx`

- Sheet：`工作表1`
  - 第 1 行：年級（P1/P2...）
  - 第 2 行：班名 + `特別任務`
  - 每一日輸出 **兩行**：
    - 上行：主題（帶底色）
    - 下行：老師（字串）
  - 同一日內「相同主題」會做橫向 merge（提高可讀性）

- Sheet：`總節數`
  - 每位老師：歷史 + 今次合計節數

- Sheet：`連續工作檢核`
  - 矩陣：日期 x 老師
  - 當一位老師連續工作 ≥ 5 日，該段會標紅（警示疲勞）

## 8) 已知行為／易出事位（接手者要留神）

- **老師代號解析**：
  - 依賴「字串長度」推斷共享/單人，對 2 字代號可能錯
  - 用 `includes()` 去判斷「某 cell 有冇某老師」，可能出現「代號互相包含」誤判
- **`TBD` 機制**：
  - 匯入/衝突檢查會強制把相關欄位設 `TBD`
  - Step 3 → Step 4 會阻止生成（好事，但要確保提示清晰）
- **歷史資料日期解析**：
  - `uploadHistory()` 係用正則抽數字砌 `YYYY-MM-DD`，若日期格式怪會失敗，排序/合併可能亂
- **CDN 依賴**：
  - 學校/內網封 CDN 就會壞；建議提供離線版

## 9) Roadmap（建議你完善嘅方向）

按「回報最多、風險最高」排序：

### 9.1 可靠性 / 正確性

- **老師代號改成明確格式**
  - 例如：`A`＝同人兩節；`A,B`＝分節；`A+B`＝共同
  - 或者 UI 入面直接分兩格「第1節老師 / 第2節老師」，避免猜測
- **衝突檢查全面化**
  - 班別固定老師 vs 特別任務 vs 自動分配，提供一致的 error list（唔只 alert）
- **日期契約**
  - 匯入/匯出日期格式固定成 `YYYY-MM-DD` 放 hidden 欄（或者另 sheet），避免用文字推斷

### 9.2 使用體驗

- **自動儲存/還原**
  - 用 `localStorage` 儲存 state（班級/老師/日期/topics/任務）
- **離線版**
  - 把 Vue/Tailwind/ExcelJS/FileSaver 打包進檔案（或改成 Vite 專案再 build）
- **更清晰嘅 TBD 導航**
  - 已有「跳至下一個 TBD」，可以加「TBD 清單 + 一鍵跳轉」

### 9.3 可維護性（長遠）

- **由單檔 HTML 拆成專案**
  - Vue + Vite，拆 `components/`、`services/excel.ts`、`scheduler/` 等
  - 加簡單單元測試：`getTeacherSlots`、衝突判斷、負荷計算、疲勞 streak
